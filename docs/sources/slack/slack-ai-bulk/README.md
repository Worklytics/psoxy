# Slack AI Analytics Bulk

**Connector ID:** `slack-ai-analytics-bulk`

**Availability:** Alpha

Psoxy can pseudonymize Slack AI Snapshot data for ingestion into Worklytics. Each export captures usage metrics (such as MAU and 30/90-day look-backs) as of a given date.

See [Slack AI Snapshot connector](https://docs.worklytics.co/knowledge-base/connectors/slack/slack-ai-snapshot) in the Worklytics knowledge base for export and connection instructions.

## Instructions to Connect

1. Add `slack-ai-analytics-bulk` to `enabled_connectors` in `terraform.tfvars`, then run `terraform apply`.
2. Review your Terraform output; find the `-input` bucket name for the connector.
3. Upload Slack AI Analytics CSV files to the `-input` bucket (filenames should include a timestamp in `yyyyMMddHHmmss` format). See [slack-ai-analytics-sample.csv](slack-ai-analytics-sample.csv) for an example export shape.
4. Create the **Slack AI Analytics - Bulk via Psoxy** connection in Worklytics; see the TODO file generated by `terraform apply`.

The default rules pseudonymize `USER_EMAIL` (required row key), optionally pseudonymize `USER_ID` and `USERNAME` when present, and redact `PRIMARY_OWNER_EMAIL`, `PRIMARY_OWNER_EMAIL_DOMAIN`, and `FULL_NAME`.
