# Gong Bulk

Psoxy can pseudonymize Gong bulk export data for ingestion into Worklytics.

See [https://docs.worklytics.co/knowledge-base/connectors/bulk-data/gong-bulk](https://docs.worklytics.co/knowledge-base/connectors/bulk-data/gong-bulk)

Data is exported from Gong's [Forecast and Gong User Tables](https://help.gong.io/docs/forecast-and-gong-user-tables)
feature as CSV files that must be uploaded periodically to the proxy input bucket.

## Data Inventory

- [USERS](#users)
- [USER_CONVERSATION_GONG_ACTIVITIES](#user_conversation_gong_activities)
- [FORECAST_SUBMISSIONS_HISTORY](#forecast_submissions_history)

### USERS

Per-user profile and configuration snapshot:

| Field Name | Type | Description |
|---|---|---|
| ACTIVE | Boolean | Whether the user is active |
| EMAIL_ADDRESS | String | The user's email address |
| FIRST_NAME | String | The user's first name |
| HOME_WORKSPACE_ID | String | The ID of the user's home workspace |
| LAST_NAME | String | The user's last name |
| LICENSED | Boolean | Whether the user has a paid license |
| LOCALE | String | The user's locale setting |
| MANAGER_ID | String | The ID of the user's manager |
| TIME_ZONE | String | The user's timezone (TZ format) |
| USER_ID | String | The unique ID of the user in Gong |
| SHOULD_IMPORT_TELEPHONY_CALLS | Boolean | Whether external telephony calls should be auto-imported |
| SHOULD_RECORD_WEB_CONFERENCE_CALLS | Boolean | Whether web conference calls should be auto-recorded |
| SHOULD_SYNC_EMAILS | Boolean | Whether emails should be automatically imported |
| TITLE | String | The user's job title |
| VALID_FROM_DATE_TIME | Timestamp | (SCD 2.0) The date this record was created |
| VALID_TO_DATE_TIME | Timestamp | (SCD 2.0) The date this record was changed |
| IS_DELETED | Boolean | Whether the user was deleted in Gong |
| ETL_MODIFIED_DATETIME | Timestamp | The date and time the data was changed |
| ROW_ID | String | The row identifier |

See [gong-bulk-rules.yaml](gong-bulk-rules.yaml) for pseudonymization and redaction rules.

### USER_CONVERSATION_GONG_ACTIVITIES

Per-user conversation activity records:

| Field Name | Type | Description |
|---|---|---|
| CONVERSATION_KEY | String | The ID of the conversation |
| GONG_ACTIVITY_DATETIME | Timestamp | The date and time of the activity |
| GONG_ACTIVITY_TYPE | String | Activity type (e.g., `listened_to_call`, `gave_feedback`, `filled_scorecard`, `gave_comment`, `feedback_marked_given`, `feedback_marked_received`) |
| USER_ID | String | The ID of the user who had the activity |
| ETL_MODIFIED_DATETIME | Timestamp | The date and time the data was modified |
| ROW_ID | String | The row identifier |

See [gong-bulk-rules.yaml](gong-bulk-rules.yaml) for pseudonymization and redaction rules.

### FORECAST_SUBMISSIONS_HISTORY

| Field Name | Type | Description |
|---|---|---|
| FORECAST_AMOUNT_UNIT | String | The unit the forecast was made in |
| FORECAST_CATEGORY_NAME | String | Category per forecast board values |
| IS_TEAM | Boolean | True indicates a team forecast |
| LINE_OF_BUSINESS | String | Line of business name |
| LINE_OF_BUSINESS_ID | String | Gong ID for the business line |
| NOTES | String | User-provided forecast notes |

| PERIOD_TYPE | String | Either `MONTH` or `QUARTER` |
| PERIOD_YEAR | Decimal | Fiscal year covered |
| SUBMIT_DATE | Timestamp | Submission timestamp |
| SUBMIT_BY_USER_ID | String | Gong ID of the user who submitted the forecast |
| USER_ID | String | Gong ID of the user the forecast is for |
| VALUE | Decimal | Forecast numerical value |
| WEEK | Decimal | Quarter week the forecast was submitted |
| IS_DELETED | Boolean | Whether this record was deleted |
| ETL_MODIFIED_DATETIME | Timestamp | The date and time the data was modified |
| ROW_ID | String | The row identifier |

See [gong-bulk-rules.yaml](gong-bulk-rules.yaml) for pseudonymization and redaction rules.

## Instructions to Connect

1. Copy [gong-bulk-rules.yaml](gong-bulk-rules.yaml) to your Terraform working directory as `gong-bulk-rules.yaml`.
2. Add `gong-bulk` to your `custom_bulk_connectors` in `terraform.tfvars` then run `terraform apply`.
3. Review your Terraform output; find the `-input` bucket name for your connector.
4. Export the `USERS`, `USER_CONVERSATION_GONG_ACTIVITIES`, and `FORECAST_SUBMISSIONS_HISTORY` tables from Gong (see [Gong documentation](https://help.gong.io/docs/forecast-and-gong-user-tables)).
5. Upload each CSV file to the `-input` bucket under the path `{date}/{table}/{filename}.csv` (e.g., `2024-01-15/USERS/USERS.csv`). Files uploaded at the bucket root will not match any rule and will fail processing.
6. Create a `Bulk Import - Psoxy` connection in Worklytics with `gong-bulk` as parser; see the `TODO 3 - connect gong-bulk` file generated by `terraform apply`.
7. Repeat steps 4–5 periodically to keep data up to date.

```hcl
custom_bulk_connectors = {
    "gong-bulk" = {
        source_kind               = "gong",
        worklytics_connector_id   = "bulk-import-psoxy"
        worklytics_connector_name = "Bulk Import - Psoxy"
        display_name              = "Gong Bulk"
        rules_file                = "gong-bulk-rules.yaml"
        settings_to_provide = {
            "Parser" = "gong-bulk"
        }
    }
}
```

The `gong-bulk-rules.yaml` applies a single rule set to all CSV files, pseudonymizing and redacting columns by name regardless of which table the file comes from. See [gong-bulk-rules.yaml](gong-bulk-rules.yaml) for the full list.
